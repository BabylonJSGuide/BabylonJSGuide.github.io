Frames of Reference

Two frames of reference are used world axes and local axes .
The frame of reference used depends whether translate and rotate take place on the world space or the local space. When world space is specified translate uses the world axes and rotate use the world axes with centre of rotation at the origin of the local axes. When local space is specified translate and rotate use the local axes with the centre of rotation at the origin of the local axes.

Translate and Rotate

Warning when translate is used with world space specified you cannot combine translations with out computing the world matrix of the mesh. When this is not done only the last translation given is used. In all other cases combinations of transformations are possible.

Since translate and rotate are about movement all examples are animated and in all examples a non symmetrical mesh, the pilot, will be used..

The Pilot

When the pilot is created its position and rotation are both set to (0, 0, 0) and all frames of reference coincide.

For translate and rotate the change can be specified to take place in the world space or in local space as follows.

The world space is set using BABYLON.Space.WORLD

The local space is set using BABYLON.Space.LOCAL

Rotate

Since rotate alters the orientation of the local axes and so can affect local translation we will conside this first. Whether in world or local space the centre of rotation is the origin of the local axes.

To rotate a mesh an axis, angle and the space specified are needed. The axis is given as any vector(x, y, z). For convenience unit vectors in the positive directions of the x, y and z axes are pre-defined as the constants BABYLON.Axis.X, BABYLON.Axis.Y and BABYLON.Axis.Z respectively.

pilot.rotate(BABYLON.Axis.Y, Math.PI / 2, BABYLON.Space.WORLD);

pilot.rotate(new BABYLON.Vector3(-1, 3, -10), 7 * Math.PI / 12, BABYLON.Space.LOCAL);

Translate

To translate a mesh a vector, distance and space need to be specified. The vector is the direction of translation, the distance * vector length gives how far to move in the given direction. you can use pilot.translate(vector, distance, space) to move the pilot the given distance in the direction of the vector based on either the world or local space. It is often useful for the vector to be a unit vector so the distance moved is precisely the distance given.

//BABYLON.Axis.Y is already a unit vector, as world space specified
//pilot will move a distance of 2 from its current position in the direction (0, 1, 0).
pilot.translate(BABYLON.Axis.Y, 2, BABYLON.Space.WORLD);

//Given vector has length sqrt(14), as local space specified 
//pilot will move a distance sqrt(14)*3 = 42 from its current position in direction (-1, 3, -2)
pilot.translate(new BABYLON.Vector3(-1, 3, -2), 3, BABYLON.Space.LOCAL);

//Since vector is normalized and local space specified 
//pilot will move a distance 6 from its current position in direction (3, 5, -2)
var direction = new BABYLON.Vector3(3, 5, -2);
direction.normalize(); // direction now a unit vector
pilot.translate(direction, 6, BABYLON.Space.LOCAL);

Unlike rotations where it is difficult to see about what axis a rotation of PI/3 about the y axis followed by a rotation of PI/5 about the x axis would combine the two it is far easier with translations. A translation of (1, 3, -5) followed by one of (2, -1, -2) is clearly (3, 2, -7) so really there is little call to write seperate translate statements. However should you wish to do so in the world space then computing the world matrix is necessary.

Playground Example (B) Translate World Space Animated with registerAfterRender -


Note the difference between example A and B, both set the position of the pilot to (2, 3, 4) within the main body of the code. In example A the pilot starts from this position whereas in example B the pilot starts from (0, 0, 0). This is because it is during the rendering process that the world matrix is computed for the mesh. Since in A the animation code is registred before the scene is rendered the world matrix has not been computed and so the position of the pilot is still set as (0, 0, 0). In B however the code is registered after the scene has been rendered and so the world matrix has been computed and the pilot position been set at (2, 3, 4);

The computation of the world matrix for a mesh can be forced as follows 

pilot.position = new BABYLON.Vector3(2, 3, 4);
pilot.computeWorldMatrix(true);

The Local Space

The local space is set using BABYLON.Space.LOCAL.

In the local space translate behaves as would a mathematical translate. That is, a sequence of translations is an accumulation of direction vectors as opposed to the setting of a position vector.

Combining translate and rotate

Whether the pilot is rotated in the world or the local space applying pilot.translate in the world space is essentially equivalent to applying pilot.position.

So what happens when pilot.translate is applied in the local space after rotations.

Firstly when rotating in the world space the word-local-axes are used for rotation and these are then also used for pilot.translate(vector, distance, BABYLON.Space.LOCAL). See Figs 8a, b

You use pilot.locallyTranslate(vector), where vector is a BABYLON.Vector3 giving the required translation.
