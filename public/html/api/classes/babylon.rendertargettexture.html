RenderTargetTexture

This Helps creating a texture that will be created from a camera in your scene. It is basically a dynamic texture that could be used to create special effects for instance. Actually, It is the base of lot of effects in the framework like post process, shadows, effect layers and rendering pipelines...

constructor

Find examples in Playground
  • new RenderTargetTexture(name: string, size: number | object | object, scene: Nullable<Scene>, generateMipMaps?: boolean, doNotChangeAspectRatio?: boolean, type?: number, isCube?: boolean, samplingMode?: number, generateDepthBuffer?: boolean, generateStencilBuffer?: boolean, isMulti?: boolean, format?: number, delayAllocation?: boolean): RenderTargetTexture

  • Overrides Texture.constructor

    Instantiate a render target texture. This is mainly used to render of screen the scene to for instance apply post processse or used a shadow, depth texture...

    Parameters

    • name: string

      The friendly name of the texture

    • size: number | object | object

      The size of the RTT (number if square, or {width: number, height:number} or {ratio:} to define a ratio from the main scene)

    • scene: Nullable<Scene>

      The scene the RTT belongs to. The latest created scene will be used if not precised.

    • Optional generateMipMaps: boolean

      True if mip maps need to be generated after render.

    • Optional doNotChangeAspectRatio: boolean

      True to not change the aspect ratio of the scene in the RTT

    • Optional type: number

      The type of the buffer in the RTT (int, half float, float...)

    • Optional isCube: boolean

      True if a cube texture needs to be created

    • Optional samplingMode: number

      The sampling mode to be usedwith the render target (Linear, Nearest...)

    • Optional generateDepthBuffer: boolean

      True to generate a depth buffer

    • Optional generateStencilBuffer: boolean

      True to generate a stencil buffer

    • Optional isMulti: boolean

      True if multiple textures need to be created (Draw Buffers)

    • Optional format: number

      The internal format of the buffer in the RTT (RED, RG, RGB, RGBA, ALPHA...)

    • Optional delayAllocation: boolean

      if the texture allocation should be delayed (default: false)

    Returns RenderTargetTexture

getCustomRenderList

Find examples in Playground
getCustomRenderList: function

Use this function to overload the renderList array at rendering time. Return null to render with the curent renderList, else return the list of meshes to use for rendering. For 2DArray RTT, layerOrFace is the index of the layer that is going to be rendered, else it is the faceIndex of the cube (if the RTT is a cube, else layerOrFace=0). The renderList passed to the function is the current render list (the one that will be used if the function returns null). The length of this list is passed through renderListLength: don't use renderList.length directly because the array can hold dummy elements!

Type declaration
